> Generated by party-mode — 2026-05-25

## 1. Objective

Xây dựng Rust CLI `pnotes` để quản lý project-level execution notes trong `mini-task` repo. Cho phép agent/human recall context từ các execution trước mà không cần đọc toàn bộ archive, không phụ thuộc memory ngoài repo.

---

## 2. Final Decision

Implement `pnotes-cli/` làm standalone Rust binary với 5 MVP commands: `init`, `add continuity`, `recall`, `show`, `guide`. Recall scoring dùng cả exact match (+5) lẫn prefix match (+4) theo spec. Frontmatter validation: hard fail khi `add` thiếu required field, warn+skip khi `recall` gặp note lỗi.

## 4. Implementation Scope

**In Scope:**
- `pnotes init` — tạo `.project-notes/notes/` directory
- `pnotes add continuity` — tạo note file với frontmatter pre-filled từ CLI args, template body rỗng
- `pnotes recall` — on-demand markdown scan + scoring, trả top N notes
- `pnotes show <id>` — in full note content
- `pnotes guide` — in workflow + command reference
- `.project-notes/README.md` — schema docs + build instructions
- `.agents/skills/project-notes/SKILL.md` — adoption mechanism cho agents
- `bin/pnotes` — compiled Linux/WSL2 binary, committed vào repo

**Out of Scope:**
- Edit, delete, update notes
- `pnotes list`, `pnotes rebuild`
- Embedding, semantic search, SQLite, daemon
- Interactive prompts hoặc wizards
- index.json committed
- Cross-platform binary
- Modification bất kỳ file Node.js hiện tại trong repo

## 6. Technical Contracts

### Data / State Contract

**Frontmatter schema (YAML, required fields):**
```yaml
id: "{YYYY-MM-DD}-{task-slug}"          # required, auto-generated
type: "continuity"                        # required, hardcoded MVP
task: "{task-slug}"                       # required, từ --task arg
created_at: "{YYYY-MM-DD}"              # required, auto từ current date
signal: "{one-line summary}"             # required, từ --signal arg
```

**Frontmatter schema (optional fields):**
```yaml
run: "{run-id}"                          # optional, từ --run arg (có thể bỏ qua MVP nếu không có arg)
handoff: "{relative-path}"               # optional, từ --handoff arg
areas:                                   # optional, từ --area args (có thể nhiều)
  - "src/session-manager"
tags:                                    # optional, từ --tag args (có thể nhiều)
  - "bug"
read_when:                               # optional, agent tự điền trong body
  - "{trigger condition}"
supersedes: []                           # optional, list ids bị supersede
```

**Body template (6 sections bắt buộc — agent tự điền):**
```markdown
## task
{task-slug} — {run-id} — {handoff-link}

## deviations
None

## traps
None

## dead_ends
None

## validation_delta
As expected

## next_agent_hint
See Handoff
```

**Immutability rules:**
- Không có update/delete command
- Note file không được sửa sau khi tạo
- Nếu cần supersede: tạo note mới với `supersedes: [old-id]`

**Note filename pattern:** `.project-notes/notes/{YYYY-MM-DD}-{task-slug}.md`

**Collision handling:** Nếu file đã tồn tại (cùng date + task-slug), append suffix `-2`, `-3`, etc.

### Action / Guard Contract

**`pnotes init`:**
- Nếu `.project-notes/notes/` chưa tồn tại → tạo dir, exit 0, print `"Initialized .project-notes/"`
- Nếu đã tồn tại → exit 0, print `"Already initialized"`

**`pnotes add continuity [--task <slug>] [--area <path>]... [--tag <tag>]... [--signal "<text>"] [--handoff <path>]`:**
- **Hard fail (exit 1) nếu thiếu:** `--task` hoặc `--signal`
- Error message rõ: `"Error: --task and --signal are required"`
- Nếu `.project-notes/notes/` chưa tồn tại → hard fail với message `"Run 'pnotes init' first"`
- Khi thành công: tạo file, print `"Created: .project-notes/notes/{id}.md"`

**`pnotes recall [--area <path>] [--tag <tag>] [--task <slug>] [--limit <n>]`:**
- Default `--limit`: 3
- Scan tất cả `.md` files trong `.project-notes/notes/`
- Nếu note có YAML frontmatter lỗi → print warning tới stderr, skip note, tiếp tục, exit 0
- Scoring algorithm (cộng dồn, không exclusive):
  ```
  +5  area exact match (sau normalization)
  +4  area parent/child prefix match (sau normalization)
  +3  task exact match
  +2  tag match (mỗi tag match +2)
  +2  text match trong `signal` hoặc `read_when` (substring, case-insensitive)
  +1  recency boost (newer = +1)
  ```
- Tie-break: `created_at` DESC (newer first)
- Nếu không có filter args → return top N notes theo recency
- Nếu không có notes → print `"No notes found"`, exit 0

**`pnotes show <id>`:**
- Tìm file `{id}.md` trong `.project-notes/notes/`
- Nếu không tìm thấy → exit 1, stderr `"Note not found: {id}"`
- Nếu tìm thấy → print full file content

**`pnotes guide`:**
- Print hardcoded text: workflow + command syntax + decision tree (khi nào dùng recall vs add)
- Exit 0

### Path Normalization (cho areas matching):
```
1. Trim trailing "/"
2. Convert to lowercase
3. So sánh: exact = (a == b), prefix = a.starts_with(b) || b.starts_with(a)
```

Ví dụ:
- note area `src/session-manager`, query `src/` → `src/session-manager`.starts_with(`src`) → prefix match +4 ✅
- note area `src/`, query `src/session-manager` → `src/session-manager`.starts_with(`src`) → prefix match +4 ✅
- note area `src/session-manager`, query `src/session-manager` → exact match +5 ✅

### UI / UX Contract

**Recall output format (mỗi note):**
```
--- {id} ---
signal: {signal}
areas:  {areas.join(", ")} (hoặc "(none)" nếu empty)
tags:   {tags.join(", ")} (hoặc "(none)")
task:   {task}
date:   {created_at}
```

**Ràng buộc output:** Top 3 notes phải fit trong < 500 tokens tổng cộng.

**Errors** → stderr. **Normal output** → stdout.

### API / Backend Contract

- Không có network calls, không external process
- Toàn bộ là file I/O trên `.project-notes/notes/*.md`
- Parser: dùng `serde_yaml` để parse YAML frontmatter block (giữa `---` delimiters)
- Nếu `serde_yaml` quá nặng cho MVP, alternative: simple line parser (key: value per line)
- Không có async — sync I/O đủ cho < 50 notes
